# 🎮 조종기 실시간 파라미터 튜너 사용 설명서

`tools/param_tuner.py` — 주행 중 게임패드로 핵심 파라미터를 실시간 조절하는 도구.
**tuning 브랜치 전용.**

## 핵심 원칙

- **params.yaml 은 절대 수정하지 않는다.** 돌고 있는 노드의 메모리 값만 바꾼다.
- 노드를 재시작하면 params.yaml 의 원래 값으로 돌아간다 (튜너 조정값은 휘발성).
- 모든 변경은 시각과 함께 로그로 남고, 종료 시 최종 값이 YAML 로 출력된다.
- 채택 여부는 사람이 결정: 마음에 들면 최종 YAML 을 params.yaml 에 직접 복사 → 커밋.

## 실행 방법

```bash
# 터미널 1 — 주행 스택 (먼저! 튜닝 대상 노드들이 여기서 뜬다)
mlane          # 또는 mauto

# 터미널 2 — 튜너
mtune          # = ms && python3 ~/ai_moon_ros2/tools/param_tuner.py
```

조종기(TGZ/Xbox 호환)는 USB 로 연결하고 전원을 켠 상태여야 한다.

### 시작 직후 확인할 것 ✅

"--- 현재 값 ---" 목록에서 **모든 값이 숫자**여야 정상.
`steer_trim = ?` 처럼 `?` 가 보이면 그 노드(control_node)가 안 떠 있는 것 —
E-STOP 포함 아무것도 동작하지 않으니 터미널 1부터 확인할 것.

## 조작법

**🎛 튜닝 모드 (기본)**

| 입력 | 기능 |
|------|------|
| 십자키 ↑ / ↓ | 튜닝할 파라미터 선택 (목록 순환) |
| **R1** / **L1** | **+큰 스텝** / **−큰 스텝** (십자키 →/← 도 동일) |
| **R2** / **L2** (트리거) | **+미세 스텝** / **−미세 스텝** (큰 스텝의 1/5) |
| **A** | 🕹 수동 조종 모드로 전환 |
| **B** | ⛔ E-STOP 토글 — 즉시 정지 + 바퀴 자동 정렬 ↔ 다시 누르면 주행 재개 |
| SELECT | 현재 값 전체 출력 |
| Ctrl+C (터미널) | 종료 → 최종 YAML 요약 출력 |

> 오른손(R) = 늘리기, 왼손(L) = 줄이기. 검지(숫자 1) = 큰 폭, 중지(트리거 2) = 미세.

**🕹 수동 조종 모드 (A 로 전환 — 자율주행 명령보다 우선)**

| 입력 | 기능 |
|------|------|
| 왼쪽 스틱 ↑↓ | 전진 / 후진 (배율 곱해짐, 시작 0.30) |
| 오른쪽 스틱 ←→ | 조향 |
| **R1** / **L1** | 수동 속도 배율 ±0.02 (0.05~0.5 클램프) — 파라미터 조절은 잠김 |
| **A** | 튜닝 모드 복귀 — 중립 1회 발행 후 0.4초 내 자율이 인계 |
| **B** | ⛔ E-STOP (수동보다도 우선. 해제 시 스틱값 리셋되어 튀지 않음) |

용도: 정지 후 차를 시작점으로 몰고 가기, 하드웨어 감각 확인 등.
방향이 반대면 param_tuner.py 상단 `MANUAL_INVERT_*` 를 True 로.

E-STOP 중에도 카메라/차선 인식/플래너는 계속 돈다. control_node 만 스로틀을
차단하므로, 정지 상태에서 파라미터를 바꾸고 B 로 재개하면 즉시 새 값으로 주행한다.
**추천 루프: B(정지) → 값 조절 → B(재개) → 관찰.**

## 튜닝 가능한 파라미터 (선택 순서대로)

| # | 표시 이름 | 파라미터 | 대상 노드 | 스텝 (미세) | 범위 | 효과 |
|---|----------|----------|-----------|------------|------|------|
| 1 | 조향 게인 | `steering_gain` | main_planner | 0.0005 (0.0001) | 0.001~0.008 | ↑ 코너 잘 돎 / 직선 지그재그, ↓ 반대 |
| 2 | 주행 속도 | `speed_safe` | lane_detection | 0.01 (0.002) | 0~0.5 | 기본 주행 속도 (출발 문턱 ~0.3 유의) |
| 3 | 직진 트림 | `steer_trim` | control_node | 0.01 (0.002) | −0.5~0.5 | 하드웨어 조향 쏠림 보정 |
| 4 | ld(빨간점y) | `sw_circle_height` | lane_detection | 10 (2) | 80~459 | ↑값 = 가까이 봄(차분/늦음), ↓값 = 멀리 봄(빠름/출렁) |
| 5 | 윈도우 폭 | `sw_margin` | lane_detection | 5 (1) | 20~150 | 추적 윈도우 반폭 — 급커브 추적력 |
| 6 | 차선 폭 | `sw_road_width` | lane_detection | 0.01 (0.002) | 0.30~0.70 | 한쪽 차선→중심 복사 거리 |
| 7 | 초기창 반폭 | `sw_win_half` | lane_detection | 10 (2) | 60~320 | 초기 탐색창 폭 (145/495 중심) |
| 8 | 초기창 높이 | `sw_win_h1` | lane_detection | 20 (4) | 150~460 | 초기 탐색창 윗변 y (작을수록 큰 창) |
| 9 | 윈도우 개수 | `sw_nwindows` | lane_detection | 1 (1) | 8~24 | 추적 높이 = 개수×20px. ⚠️ ld(빨간점) 높이를 커버해야 점이 찍힘: 개수×20 ≥ 480−ld |
| 10 | 스로틀 게인 | `throttle_gain` | main_planner | 0.05 (0.01) | 0.5~1.5 | 전체 속도 스케일 (speed × gain) |
| 11 | 최대 스로틀 | `max_throttle` | control_node | 0.05 (0.01) | 0.2~0.8 | 최종 속도 상한 — 아무리 올려도 이 위로 안 나감 |
| 12 | fast 속도 | `speed_fast` | lane_detection | 0.01 (0.002) | 0~0.6 | version: fast 모드일 때의 주행 속도 |

모든 값은 범위 밖으로 못 나가게 클램프된다.

## 커브 이탈 튜닝 추천 순서

한 번에 하나만 바꾸고, 같은 커브에서 재현 → 8080 디버그로 확인.

1. **인지부터** — 디버그 화면에서 윈도우 계단이 커브의 선을 끝까지 무는가?
   - 계단이 선에서 갈라짐 → **⑤ 윈도우 폭** ↑
   - 커브 진입 때 빈 박스만 뜸 → **⑦ 초기창 반폭** ↑ 또는 **⑧ 초기창 높이** ↓
2. **중심 추정** — 차를 차선 중앙에 두고 빨간 점이 x=320 근처인가?
   - 치우침 → **⑥ 차선 폭** 조정
   - 코너 반응이 늦음/출렁임 → **④ ld** ↓(멀리) / ↑(가까이)
3. **조향** — 인지가 정확한데 못 따라감 → **① 조향 게인** ↑ (직선 지그재그 시작되면 한 스텝 back)
4. **속도** — 위로도 해결이 안 되면 **② 주행 속도** ↓ 가 마지막 카드

경험칙: 커브 이탈의 대부분은 ⑤(윈도우 폭)와 ①(게인)에서 잡힌다.

## 로그와 최종 요약

- 실시간 로그: 터미널 + `~/ai_moon_ros2/tune_logs/tune_YYYYMMDD_HHMMSS.log`
  - 선택/변경/E-STOP 이 전부 시각과 함께 기록됨 → "몇 시에 뭘 바꿨더니 좋아졌지?" 복기 가능
- Ctrl+C 종료 시:

```
==============================================
 최종 튜닝 요약 — params.yaml 에 붙여넣기
==============================================
main_planner:
  ros__parameters:
    steering_gain: 0.003
lane_detection_node:
  ros__parameters:
    speed_safe: 0.22
    sw_circle_height: 250
    ...
```

이 블록을 params.yaml 의 해당 섹션에 **값만** 옮겨 적으면 확정된다
(섹션 헤더는 이미 params.yaml 에 있으므로 값 줄만 복사).

## 문제 해결

| 증상 | 원인 | 조치 |
|------|------|------|
| 값이 전부 `?` + ⚠️ 경고 | 주행 스택이 안 켜짐 | 터미널 1 에서 `mlane` 먼저 |
| `steer_trim = ?` 만 | control_node 부재/사망 | `mlanenc` 로 켰는지, 터미널 1 에러 확인 |
| 버튼 눌러도 로그 자체가 안 뜸 | 조종기 연결/전원 | `ls /dev/input/js0` 확인, 재연결 |
| 로그는 뜨는데 차가 무반응 | 위의 `?` 상황과 동일 | 시작 시 현재 값 목록부터 확인 |
| 값 변경 로그에 "적용 실패" | 대상 노드가 도중에 죽음 | 터미널 1 로그 확인 후 재시작 |
